【数据结构】并查集

发表于 2022-08-03 分类于 数据结构

相关概念：

  • 并查集是一种树形的数据结构，用于处理集合相关的问题。
    • 通常集合中的元素都不具备层次关系，而是因为共通的性质而被划分在一起。
    • 当存在多个集合，而又需要判断任意两个元素是否处在同一个集合中时，就需要使用并查集来维护集合元素之间的关系。
  • 并查集有两个主要的操作：
    1. 合并操作：将两个子集合并成一个子集。
    2. 查找操作：确定某个元素处在哪个集合。
  • 一般用固定大小的数组来实现并查集，但可以用森林的模型来解释并查集的操作。【森林即多棵树】
    • 假设题目中元素只有 m 个，则并查集底层只需维护一个 m 大小的数组 parent 即可。【parent 数组保存对应元素的父节点】
    • 对于合并操作的解释：将两个元素合并在同一个子集，就相当于将两棵树合并成一棵树。
    • 对于查找操作的解释：要确定两个元素处于同一个集合只需要确定两个元素是否处于同一棵树即可。

核心思路：

  • 并查集的实现非常简洁。
    • 初始化：设置每个元素为一个集合，即设置每个元素的父节点为自己。
    • 查找：找到元素所在集合的根，即所在树的根。【从代码中看出，find 函数会递归调用，直到找到集合的根】【此处的 find 函数已经进行路径压缩，简单来说，就是把当前搜索路径上的所有元素都连接到最顶部的根上去，这样每一棵树的高度都不会过分高】
    • 合并：分别找出两个元素的根进行连接即可。

    1. 假设有 5 个元素，则初始化一个大小为 5 的 parent 数组：
    2. 假设元素 0 和元素 1 有关系，则调用 union(0, 1)，注意此时 find(0) 或 find(1) 都是简单地返回 0 或 1：
    3. 假设元素 0 分别和元素 2 、元素 3 有关系，则分别调用 union(2, 0) 和 union(3, 0)，注意 find(0) 会返回 1：
  • 最后要提及的是关于并查集的一些优化：
    • 可以注意到，上面例子的最后结果中留下了两棵树（即两个集合），其中 root 1 体型较大，root 4 体型较小，如果后续调用 union(1, 4) 则会把大树放置到小树底下（这样显然是不够平衡的），然而调用 union(4, 1) 就会避免前面的情况。【这样说明了不同的参数调用顺序会给 union 带来不同的结果】
    • 有两种优化思路，一种是 with size，一种是 with rank。
    • 简单来说，with size 方法就是并查集中维护一个 size 数组（记录元素所在树的大小），在 union 操作中进行更新；同理，with rank 方法就是维护一个 rank 数组（记录元素所在树的高度），同样是在 union 操作中进行更新。

【leetcode】【剑指offer Ⅱ】002.二进制加法

发表于 2022-08-03 分类于 刷题 ， 剑指offer Ⅱ

问题描述：

  • 给定两个 01 字符串 a 和 b，计算它们的和并以二进制字符串的形式输出。
    • 输入字符串非空且只包含 0 和 1。

核心思路：

  • 该题是 leetcode 第 2 题两数相加的二进制字符串版本。
  • 模拟进位加法即可，算法步骤如下：
    1. 设置进位 ca 为 0，并初始化结果字符串 ans 为空字符串。
    2. 从后往前遍历两个字符串，只要有一个字符串未走完全程则继续遍历。
      1. 遍历过程中，分别取得字符串 a 和 b 在各自索引的数值（如果字符串 a 或 b 已走完全程就直接将对应数值置为 0）。
      2. 将两个数值与进位 ca 进行相加，根据相加结果更新 ans 和 ca。
    3. 遍历结束后，判断进位 ca 是否为 1，如果为 1 则需要更新 ans。
    4. 最后将 ans 进行 reverse 操作即可。

问题描述：

  • 给定两个整数 a 和 b，求它们的除法的商 a/b，要求不得使用乘号、除号以及求余符号。
    • 整数除法的结果应当截去其小数部分。【即向下取整】
    • 假设环境只能存储 32 位有符号整数，其数值范围是 $[−2^{31}, 2^{31}−1]$。
    • 如果除法结果溢出，则返回 $2^{31}−1$。

核心思路：

  • 最朴素的方法是很直观的，首先假设 a 和 b 均为正整数，一直使用除数 a 减去被除数 b，直到 a < b，其中记录并更新进行减操作的次数，相当于以步进的方式更新商：

    • 优化后不再采取步进的方式，而是通过倍增被除数 b 来找到商：
  • 目前的思路只考虑 a 和 b 均为正整数的情况，且没有考虑很多的溢出情况，后续只需要对所有可能溢出的地方进行处理即可。【但通常处理溢出都很繁琐】

【leetcode】【2022/8/2】622. 设计循环队列

发表于 2022-08-02 分类于 刷题 ， leetcode每日一题

问题描述：

  • 设计循环队列。
    • 循环队列是一种线性数据结构，其操作表现基于 FIFO（先进先出）原则并且队尾被连接在队首之后以形成一个循环。
  • 该实现类 MyCircularQueue 支持如下操作：
    • MyCircularQueue(k)：构造器，k 为队列的长度。
    • Front()：从队首获取元素，如果队列为空，返回 -1。
    • Rear()：获取队尾元素，如果队列为空，返回 -1。
    • enQueue(value)：向循环队列插入一个元素，如果成功插入则返回真。
    • deQueue()：从循环队列中删除一个元素，如果成功删除则返回真。
    • isEmpty()：检查循环队列是否为空。
    • isFull()：检查循环队列是否已满。

核心思路：

  • 初始的模拟思路很直观：
    • MyCircularQueue 中包含三个成员，vec（固定大小的数组）、 head（队列头位置索引）和 tail（队列尾位置索引）。【注意 tail 代表尾后位置】
    • 初始化 MyCircularQueue(k) 中将 vec 置为大小为 k 的数组，再将 head 和 tail 均置为 0。
    • 很自然地，初始化后 head == tail == 0，此时循环队列为空，则至少说明 isEmpty() 可以根据 head 和 tail 的值是否相等来返回结果。
    • 向循环队列插入或删除操作也不难推导出来，插入操作在设置值的同时把 tail 后移一位，删除操作直接将 head 后移一位。【插入操作需要提前判断循环队列是否已满，删除操作需要提前判断循环队列是否为空】【head 的后移一位操作在遇到边界的时候需要重新回到索引 0，所以后移操作做一个取模操作即可，即 (head+1)%sz（其中 sz 为 vec.size()）】【tail 的后移操作同理】
    • Front() 首先判断队列是否为空，不为空则返回 head 索引上的值。
    • rear() 首先判断队列是否为空，不为空则返回 (tail+sz-1)%sz 索引上的值（其中 sz 为 vec.size()）。
    • 到目前为止，这个思路的方向还算正确，但是在判断循环队列是否为满时会出现问题：
      • 如下图所示，循环队列在 enQueue(3) 后使得循环队列为满，但此时再次出现 head == tail：
      • 这种情况说明如果按目前的假设， isEmpty() 的实现和 isFull() 的实现会产生矛盾。
  • 为了避免判空和判满的矛盾，有两种方法：【两种方法均是延续原来的思路，对原来的数据结构进行补充】
    1. 设置一个 bool 类型值 flag，flag 在 enQueue(value) 和 deQueue() 中维护更新，则判满操作就可以根据 flag 值的不同与判空操作进行区分。
    2. 在初始化中将 vec 设置为大小为 k+1 的数组，则判满操作可以设置为返回 head == (tail+1)%capacity，其他操作仍然和上述思路保持一致。【后面代码实现是依据方法二】

【leetcode】【2022/7/30】952. 按公因数计算最大组件大小

发表于 2022-07-30 更新于 2022-08-01 分类于 刷题 ， leetcode每日一题

问题描述：

  • 给定一个数组 nums，当 nums[i] 和 nums[j] 有一个大于 1 的公因数时，认为两个数存在有一条边，返回图中最大连通组件的大小。
    • 满足 $ 1 <= nums.length <= 2 * 10^4 $。
    • 满足 $ 1 <= nums[i] <= 10^5 $。
    • nums 中所有值都不同。

核心思路：

  • 第一反应是并查集 + gcd：
    • 以 nums.size() 作为并查集的大小，尝试连通 nums 中各个数值。
    • 用 gcd 来求得两个数的最大公约数，判断结果是否大于 1。
    • 但用这个方法进行模拟会超时，只通过了部分案例。【关键应该在于 nums[i] 的取值范围达到 $10^5$，假设是两个很大的质数，用来求 gcd 的过程就会很耗时间】
  • 官方题解中的思路也是利用并查集，但判断连通组件大小之前会将 nums[i] 与各自的因数先连通。
    • 以 max(nums) + 1 作为并查集的大小。
    • 接着遍历 nums 数组，将 nums[i] 的因子与 nums[i] 进行连通。【将 sqrt(nums[i]) 往下的因子进行连通即可（即进行 union 操作），因为因子是成对的，每次能将 nums[i] 分别与两个数连通】
    • 最后就是遍历 nums 数组，在并查集中搜索 nums[i] 的 root 进行大小上的累加，过程中确认最大值即可。

【图论】拓扑排序

发表于 2022-07-28 更新于 2022-08-02 分类于 算法 ， 图论

相关概念：

  • 拓扑排序，就是在有向图中找到一条拓扑序列，序列中排列在前的节点相较于排列在后的节点有更高的优先级。
    • 首先要理解优先级，优先级是有向图中边的方向性带来的结果，如 a -> b 表示节点 a 指向节点 b，则可以认为 a 的优先级优于 b，在特定语境下可以说 a 必须在 b 之前完成，如下图所示，该有向图的拓扑排序结果为 14325 或 41325。
    • 拓扑排序算法可以用来判断有向图是否有环，可以生成拓扑序列。
      • 网络中可能存在多个拓扑排序结果。
      • 如果网络中不存在环，则结果中会包括所有节点。
      • 但如果网络中存在环，则结果中可能会缺少某些节点。
  • 图可以有不同的表现形式，如邻接矩阵表示形式和邻接表表示形式，有向图的邻接表表示形式。
    • 简单来说，假设图中有 m 个节点，邻接表就是维护一个 m 大小的数组 vec，其中数组元素为当前节点的外接节点链表。【例如在网络中 a -> b 且 a -> c，则数组中 a 节点对应的位置就保存一个链表 b => c，相当于 vec(a) => b => c （此处用 => 表示链表指针，b 和 c 在链表中的顺序不分先后）】
    • 也就是说，数组中第 i 个元素（记住数组元素为链表）保存的是网络中第 i 个节点的 one-hop neighbor。【即节点跳一步就可以达到的邻居节点】
  • 进行后续算法的介绍前，还需要理解节点的入度。
    • 当前节点的入度，指的就是网络中有多少节点直接指向当前节点。【如上图所示，节点 3 的入度为 2 】
    • 可用一个数组统计每个节点的入度。

核心思路：

  • 拓扑排序问题有两种解决方案：Kahn 算法和 dfs 着色解法。
  • Kahn 算法的核心思想就是每次删除一个当前入度为 0 的节点，并删除该节点对链出邻居的影响（即其指向邻居的入度数减一）。
    • Kahn 算法需要图的邻接表形式。
    • 可以用 stack 或 queue 来保存当前入度为 0 的节点。【即用 stack 或 queue 维护当前入度为 0 的点集】
    • 用上面的图来理解，Kahn 算法的第一步就是把节点 1 带来的影响消去，留下其余的节点和边作为子图进行拓扑排序：
  • dfs 着色解法即利用深度优先算法的方法以任意顺序遍历图中所有节点，但遍历的过程中需要对节点着色。
    • 不同的颜色代表访问的程度深浅，过程中遇到不同颜色的节点会有相应的处理；需要一个数组维护网络中节点的颜色，颜色的状态可以为 0、-1 和 1。
      • -1 表示访问过，如果重复遇上，则说明存在环。
    • 最后需要把结果数组进行 reverse 操作，得出来的才是拓扑序列。【因为 dfs 着色解法会优先深度搜索到优先级最低的节点，所以最先存入结果数组的就是优先级最低的节点，最后结果数组就需要翻转过来才可】
